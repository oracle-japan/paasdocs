    Werckerを使ってみよう¶

    このドキュメントは、Werckerを使用したサンプルアプリケーションを数分で開発、ビルド、デプロイするためのステップバイステップのガイドです。

    このガイドではGoを使用していますが、このチュートリアルで説明する一般的な概念は他のすべてのプログラミング言語にも当てはまります。

    要件¶

    このガイドに従うには、次のものが必要です。

    • GitHubアカウント
    • Wercker アカウント
    • Wercker CLI

    Note

    Wercker CLIは現在MacOS XとLinuxをサポートしています。プラットフォーム毎のインストール手順は、公式ドキュメントのガイドを参照ください。

    アプリケーションの準備¶

    開発を始める前に、サンプルアプリケーションをご自身のGitHubアカウントにフォークし、さらにローカル開発環境にクローンしておく必要があります。それを完了したら、プロジェクトディレクトリに移動（cd）してください。

    getting-started-golang/

    次に、アプリケーションをビルドして実行し、問題なく機能していることを確認します。

    go build

    ./getting-started-golang

    ブラウザでlocalhost:5000/cities.jsonにアクセスすると、次のjsonが表示されます。

    {"cities":["Amsterdam","Berlin","New York","San Francisco","Tokyo"]}

    アプリケーションの開発¶

    アプリケーションのセットアップが完了したので、開発を始めることができます。現在の都市リストはちょっと短いので、そこに別の都市を追加してみませんか？しかしこれを行う前に、まずプロジェクトフォルダに含まれているwercker.ymlファイルを見てみましょう。

    wercker.yml¶

    wercker.ymlは、werckerを使うために必要な唯一の設定ファイルです。その中で、アプリケーションの開発、ビルド、およびデプロイに必要なすべてのステップを定義します。まず、ファイルの上部を詳しく見てみましょう。

    # The container definition we want to use for developing our app
# アプリケーションの開発に使用するコンテナの定義
box:
  id: golang
  ports:
    - "5000"

    この最初のboxセクションでは、2つのことを定義します。 idプロパティは、プロジェクトに使用するコンテナイメージを示します。このサンプルでGoを使って開発しているので、デフォルトのgolangイメージを使用しています。この例のようにイメージ名だけを指定すると、Docker Hubから取得されます。portsプロパティは、実行中のコンテナに公開するポートを指定します。この例では、一つのポートしか必要としませんが、ここに複数のポートを指定できます。

    さて、devのセクションを見てみましょう。

    # Defining the dev pipeline
# devパイプラインの定義
dev:
  steps:
    - internal/watch:
        code: |
          go build ./...
          ./source
        reload: true

    ここでは、devパイプラインで何を実施するかを定義します。この場合、internal/watchという1つのステップが記述されています。internal/watchのステップでファイルの変更を監視し、reloadがtrueに設定されている場合は、変更がすぐに反映されるようにアプリケーションを再起動します。これは、今回のようにWebアプリケーションを開発しているときに特に便利です。ステップの実体は、werckerまたはコミュニティのいずれかが作成したbashスクリプトです。ステップについての詳細はこちらで確認することができます。

    次はこれらのステップの動きを見るので、devパイプラインを起動しましょう。

    Werckerはまずコードをチェックアウトし、続いてコンテナ環境をセットアップします。これは、コンテナがDocker Hubからpullされ、その後、チェックアウトされたコードにコンテナがアクセスできるように準備されます。続いてwercker.ymlで定義されているステップの実行を開始します。

    再度localhost:5000/cities.jsonにアクセスすると、以前と同じ出力が表示されます。実際のリロードの動きを見てみましょう。 main.go:15で新しい都市を配列に追加します。

    Cities: []string{"Amsterdam","Berlin","New York","San Francisco","Tokyo","London"}}

    変更を保存すると、アプリケーションは自動的にリロードされます。ページをリフレッシュしたら、新しい都市がそこにあるはずです！万歳！

    実際のアプリケーションの開発ではさらに多くのステップが必要です。探してみて、使いたいステップを見つけられなければ、あなた自身で作成することもできます。

    開発が完了したので、次はアプリケーションをビルドして、デプロイしていきます。

    アプリケーションのビルド¶

    まず、wercker.ymlをもう一度見てみましょう。

    # The container definition we want to use for developing our app
# アプリケーションの開発に使用したいコンテナの定義
box:
  id: golang
  ports:
    - "5000"

# Defining the dev pipeline
# devパイプラインの定義
dev:
  steps:
    - internal/watch:
        code: |
          go build ./...
          ./source
        reload: true

build:
  steps:
    - wercker/golint
    - script:
        name: go build
        code: |
          go build ./...
    - script:
        name: go test
        code: |
          go test ./...

    パイプラインのビルド¶

    次はbuildパイプラインで何が起きるかに注目しましょう。wercker/golintのステップから始まる、いくつかの新しいステップを追加しました。wercker/golintステップはコードをチェックし、良い仕事でないと判断されれば失敗させます。また、scriptステップという新しいステップも追加しました。scriptステップは、テストを実行するために使用しているインラインbashコードです。このようなステップを作成して、Werckerのリポジトリにステップを登録することで、コミュニティへ共有することができます。残りのステップはおそらく自明でしょう。ビルドとテストのためのステップです。

    ローカルでビルドすることは、いくつかの変更が加えられてコードが正しく実行されるかどうかわからないときに非常に便利です。Gitプロバイダに変更をプッシュする前に動作確認することができます。

    しかし、アプリケーションがコンパイルされ、正しく動作していることが確認されたので、次のセクションでやっているように、werckerがクラウドでアプリケーションをビルドしてデプロイするようにしましょう。

    Werckerへのアプリケーションの追加¶

    次のステップは、werckerで新しいアプリケーションを作成することです。 https://app.wercker.com/ にアクセスし、トップメニューバーから[Add application]を選択します。

    OwnerとGit Providerの選択¶

    最初に、アプリケーションの所有者を選択する必要があります。このチュートリアルでは、まず自分自身を選択してください。必要に応じて、werckerで作成した組織を選択することもできます。

    GitプロバイダとしてGitHubを選択して（BitBucketやGitLabなど、いくつかのGitプロバイダを利用可能です）、[Next]をクリックします。

    リポジトリの選択¶

    既存のリポジトリのリストが表示されます。前の手順でフォークしたgo言語のサンプルアプリケーションのリポジトリを選択し、[Next]をクリックします。

    アクセス方式の設定¶

    次のステップはリポジトリへのアクセス方式を設定することです。一番上に表示されている選択肢ではWerckerがSSHキーを使わずにコードをチェックアウトします。今回のアプリケーションはオープンソースであり公開されたアプリケーションですので、これで問題ありません。そのまま、[Next]をクリックします。

    アプリケーション詳細の確認¶

    最後に、すべての設定を確認したら、[Create]をクリックしてアプリケーションの設定を完了します。これで最初のビルドを開始できます！

    ビルドの実行¶

    うまくいきました！画面には、wercker.ymlファイルを生成するオプションが表示されているかもしれませんが、リポジトリにすでに含まれているので、今すぐビルドを開始するだけです。

    Runページに、新しいビルドがトリガーされたことが表示されます。このビルドは、ローカルで起動されたビルドとまったく同じになりますが、チームの全員にこのビルドを共有することができています。

    最後に¶

    おめでとう！あなたはWerckerを使って最初のアプリケーションをビルドしました。
