    Fn Project handson¶

    Fn ProjectsをWindowsのみで動かす¶

    Windowsのみを用いて、Fn Projects上でHello worldできるようにします

    動作条件¶

    • Windows10 Pro
    • Docker for Windowsがインストール済み

    全体の流れ¶

    大まかには、やることは以下の2つだけです 1. Fnのセットアップ(Client,Server) 1. Functionのデプロイ

    Fnのセットアップ¶

    Fn Cliのセットアップ¶

    1. 以下のURIよりexeを取得します
      https://github.com/fnproject/cli/releases/download/0.5.82/fn.exe
    2. exeを任意の場所(例：C:\fn\fn.exe)に配置し、PATHを通す
    3. コマンドプロンプト起動
    4. fn --versionで結果が返ってきたら、準備完了です

    Fn Serverのセットアップ¶

    Fn Projectsでは、Docker上のコンテナでコードを実行します
    - Windows環境では、以下のコマンドを実行しDocker上にFn Serverを起動します。 

    docker run --rm -i --name functions -v /var/run/docker.sock:/var/run/docker.sock -p 8080:8080 -v /mnt/docker/data:/app/data -v /mnt/docker/iofs:/iofs -e FN_IOFS_DOCKER_PATH=/mnt/docker/iofs -e FN_IOFS_PATH=/iofs fnproject/functions`

    ※linux環境ではfn startのみでServerの起動ができます

    セットアップの確認¶

    • 別のコマンドプロンプトを開き、fn versionで以下のような結果が返ってきた場合、
      Cli, Server共にセットアップが正常に完了しています

      C:\fn>fn version Client version is latest version: 0.5.82 Server version: 0.3.702

    contextの設定¶

    Fn Clientは、複数のコンテナに向けて利用されることを想定しているため、
    「どのコンテナに向けるか」の設定を保持しており、それを「context」と呼びます
    初回セットアップ後は、defaultのcontextを選択する必要があります

    1. fn use context default で「default」という名前のcontextを選択
    2. fn update context registry fnuser でregistryの変更
      ※fnuserの部分はなんでも構いません

    3. fn list contextsでcontextの確認をすると、以下のようになります

      C:\fn>fn list contexts CURRENT NAME PROVIDER API URL REGISTRY * default default http://localhost:8080 fnuser

    Java FDKでHello world¶

    Fn Projectsでは、コマンド1つでテンプレートのFunction/アプリケーションの作成ができます

    Function作成¶

    1. 以下のコマンドを実行し、/javafnをhttpサーブレットのトリガーとする、Javaのテンプレートfunctionを作成します
      fn init --runtime java --trigger http javafn

    2. cd javafn⇒treeでテンプレートFunctionの階層が確認できます

      C:. └─src ├─main │ └─java │ └─com │ └─example │ └─fn └─test └─java └─com └─example └─fn

    3. また、type func.yamlでテンプレートFunctionの設定ファイルが確認できます

      C:\fn\javafn>type func.yaml schema_version: 20180708 name: javafn version: 0.0.3 runtime: java build_image: fnproject/fn-java-fdk-build:jdk11-1.0.98 run_image: fnproject/fn-java-fdk:jre11-1.0.98 cmd: com.example.fn.HelloFunction::handleRequest triggers: - name: javafn type: http source: /javafn

    Functionの実行¶

    • curlまたはfn invokeを利用し、Functionを実行できます

    curl 

    curl http://localhost:8080/t/java-app/javafn

    fn invoke 

    fn invoke java-app javafn

    • 引数を与えれば、名前を変えることもできます(Windowsではcurlのみ)
      [INPUT-NAME]の部分を任意のものに変更してください

      curl -H "Content-Type: application/json" -d [INPUT-NAME] http://localhost:8080/t/java-app/javafn
