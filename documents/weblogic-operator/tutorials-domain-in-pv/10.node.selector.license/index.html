    Oracle WebLogic Operator チュートリアル¶

    ライセンス対象のノードに対するWebLogicのPodの配置¶

    このユースケースはノードに対するWebLogicのPodの割り当て で実施した個々のサーバー/Podを特定のノードに配置するケースに似ています。ここではライセンス適用範囲のユースケースにフォーカスします。

    Kubernetesはバージョン v1.13から5,000ノード以上のクラスタに対応しました。しかし、WebLogicのような特定のソフトウェアはライセンスを必要とします。nodeSelector の機能を利用してWebLogicのPodをライセンスされたワーカーノードに配置します。

    今回のハンズオンでは、WebLogicドメイン内の全てのWebLogicのPodを特定のノードにアサインする方法を学びます。

    ライセンスされたノードに対するWebLogicサーバーのPodのアサイン¶

    ノードにPodをアサインするために、カスタム・タグを利用して希望するノードにラベルを付与する必要があります。ドメイン・リソース定義に nodeSelector プロパティを定義し、ノードに適用したラベルの値を設定します。最後にドメイン設定の変更を適用します。

    最初に kubectl get node を使用してノード名を取得します。

    kubectl get node

    NAME         STATUS   ROLES   AGE     VERSION
10.0.10.2    Ready    node    7h20m   v1.14.8
10.0.10.3    Ready    node    7h16m   v1.14.8

    OKEの場合、ノード名はパブリックIPアドレスやサブネットのCIDRブロックの最初のIPアドレスになることがあります。但し、明確にはノードを特定するユニークな文字列です。

    kubectl get pod -n sample-domain1-ns -o wide を使い、Pod情報の詳細から現在のPodの配置をチェックします。

    kubectl get pod -n sample-domain1-ns -o wide

    以下のような出力が得られます：

    NAME                                                     READY   STATUS      RESTARTS   AGE     IP            NODE         NOMINATED NODE   READINESS GATES
sample-domain1-admin-server                              1/1     Running     0          11m     10.244.3.31   10.0.10.3    <none>           <none>
sample-domain1-create-weblogic-sample-domain-job-djk84   0/1     Completed   0          157m    10.244.3.8    10.0.10.3    <none>           <none>
sample-domain1-managed-server1                           1/1     Running     0          10m     10.244.2.14   10.0.10.2    <none>           <none>
sample-domain1-managed-server2                           1/1     Running     0          10m     10.244.3.32   10.0.10.3    <none>           <none>
sample-domain1-managed-server3                           1/1     Running     0          10m     10.244.2.15   10.0.10.2    <none>           <none>

    Kubernetesの結果から3つの管理対象サーバーが2つのワーカー・ノードに配置されていることが確認できます。このシナリオではすべてのPodを移動する1つのノードを選択します。

    ラベルの設定¶

    今回はライセンスされたノードを 10.0.10.2 とします。

    このノードにラベルを設定します。任意のラベル名を指定できますが、ここでは licensed-for-weblogic を使用します。ノード名を正しく設定し kubectl label nodes <nodename> <labelname>=true コマンドを実行してください。

    kubectl label nodes 10.0.10.2 licensed-for-weblogic=true
    ドメイン・リソース定義の変更¶

    定義の変更を適用する前に、一度admin-serverとmanaged-serverのpodを削除します。 以下のコマンドを実行してください。

    kubectl edit domain sample-domain1 -n sample-domain1-ns

    serverStartPolicyをNEVERに設定して、保存します。

    以下のような出力が得られればpodが削除されています。

    kubectl get po -n sample-domain1-ns -o wide

    以下のような出力が得られます：

    NAME                                                     READY   STATUS      RESTARTS   AGE     IP            NODE         NOMINATED NODE   READINESS GATES
sample-domain1-create-weblogic-sample-domain-job-djk84   0/1     Completed   0          3h3m    10.244.3.8    10.0.10.3    <none>           <none>

    再度ドメインリソースを修正します。以下のコマンドを実行してください。

    kubectl edit domain sample-domain1 -n sample-domain1-ns

    adminServer:とserverPod: エントリを見つけ、中に新しいプロパティを追加します。

    adminServer:
  [...]
  serverPod:
    nodeSelector:
      licensed-for-weblogic: true

    serverPod:
  env:
  [...]
  nodeSelector:
    licensed-for-weblogic: true

    インデントに注意してください。文法についてはサンプル domain.yaml の同じ部分のコメント内の記述でダブルチェックできます。

    最後にserverStartPolicyをIF_NEEDEDに設定して、保存します。

    Operatorは変更に従ってサーバーの再配置を開始します。繰り返し情報を更新して期待した結果が表示されるまで待ってください。

    kubectl edit domain sample-domain1 -n sample-domain1-ns

    最終的には以下のような出力が得られます：

    NAME                                                     READY   STATUS      RESTARTS   AGE     IP            NODE         NOMINATED NODE   READINESS GATES
sample-domain1-admin-server                              1/1     Running     0          4m9s    10.244.2.17   10.0.10.2    <none>           <none>
sample-domain1-create-weblogic-sample-domain-job-djk84   0/1     Completed   0          3h13m   10.244.3.8    10.0.10.3    <none>           <none>
sample-domain1-managed-server1                           1/1     Running     0          3m13s   10.244.2.18   10.0.10.2    <none>           <none>
sample-domain1-managed-server2                           1/1     Running     0          3m13s   10.244.2.20   10.0.10.2    <none>           <none>
sample-domain1-managed-server3                           1/1     Running     0          3m13s   10.244.2.19   10.0.10.2    <none>           <none>
    ラベルと nodeSelector エントリの削除¶

    ノード配置を削除するために、ノード名を適切に指定して各ノードのラベルを kubectl label node <nodename> <labelname>- コマンドを使って削除してください。

    kubectl label nodes 10.0.10.2 licensed-for-weblogic-

    ドメインリソース 内のノード配置の記述を削除、またはコメントアウトしてください。

    kubectl edit domain sample-domain1 -n sample-domain1-ns

    一度serverStartPolicyをNEVERに設定して、保存します。

    そのあと、serverStartPolicyをIF_NEEDEDに設定して、保存します。Podはスケジューラの決定に基づいて再配置/再起動されます。

    以上で、WebLogic Operatorのハンズオンは終わりです。お疲れ様でした。
